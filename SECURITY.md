# Security Policy

## Supported Versions

We support security patches for the following versions of QueueCTL:

| Version | Supported |
| :--- | :---: |
| v1.0.x  |  Yes  |
| < v1.0  |  No   |

---

## 🔒 Reporting a Vulnerability

We take the security of QueueCTL seriously. If you find a security vulnerability, please do **not** report it via public GitHub issues. Instead, report it privately to our security team.

### Vulnerability Report Channels
*   **Email**: Send a detailed report to `security@queuectl.dev`.
*   Please include a Proof of Concept (PoC) or step-by-step instructions to reproduce the issue.

### Acknowledgment & Resolution SLA
*   **Acknowledgment**: We will acknowledge receipt of your vulnerability report within **48 hours**.
*   **Updates**: We will provide weekly updates on the progress of the investigation and fix.
*   **Fix SLA**: We aim to resolve and publish a security advisory with patches within **30 days** of receiving the report.
*   **Disclosure**: A public security advisory (with CVE assignment, if applicable) will be published on GitHub once a patched release is available.
